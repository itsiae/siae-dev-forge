## 2. Decisioni architetturali (ADR)

### ADR-1: Orchestration-only gate (IaC + ETL Terragrunt)
**Decisione:** repo `*-iac` / `*-iaac` per nome OR `>50% .tf+.hcl` per content AND no runtime manifest → early-exit Block 4 "Unit tests N/A for orchestration repo. Use integration tests."
**Rationale:** dataplatform-dwh-etl pattern, conferma utente, evita pipeline futile.

### ADR-2: Manifest-root surfacing
**Decisione:** nuovo campo `manifest_root` in `stack.json` (default `"."`); per monorepo SIAE diventa `modules/service/lambda` o equivalente. `select_command.py`, `validate_env.py`, Phase 4/6 `cd $manifest_root`.
**Rationale:** 3/4 repo agent-test sono di questa famiglia; senza fix, classe intera di repo SIAE rotta.

### ADR-3: Anchored path_pattern in `_classify_priority`
**Decisione:** Pass-2 fallback matcha solo `last_2_segments` del rel_path, non l'intero path.
**Rationale:** Java `it/siae/pae/services/...` o `modules/service/lambda/src/api/...` classificano 90% file come P1 esplodendo il floor enforcement loop. Anchored match fissa il bug senza rompere parent_dirs Pass-1.

### ADR-4: Codegen file skip via marker header
**Decisione:** `priority-rules.json.generated_file_markers` (lista esplicita: `@generated`, `auto-generated`, `autogenerated`, `AUTO-GENERATED`, `DO NOT EDIT`, `DO NOT MODIFY`, `Generated by`, `@auto-generated`, `<auto-generated>`) + `_is_generated_file()` in `estimate_size.py` (sniff prime 1024 bytes). + estensione `skip_patterns` per `*.imports.{ts,js,mjs}`, `*.generated.{ts,js,py}`, `*.gen.ts`, `*.codegen.*`, `scripts/generate-*.{ts,js,mjs}`, `scripts/gen-*.{ts,js}`.
**Rationale:** codegen files (OpenAPI/Drizzle/Prisma/jOOQ/protoc) brucia iter budget; un-testabili in unit style.

### ADR-5: SKILL.md compaction 230→~90 LOC
**Decisione:** estrarre Phase-1/6 inline bash → `lib/phase{1,6}-*.sh`. Estrarre Phase-2/7 markdown → `references/phase-{2,7}-*.md`. SUPPORTING FILES table → `references/index.md`.
**Rationale:** SKILL.md e' sempre-loaded, riduce overhead di ~10K token per attivazione, allinea ad Anthropic progressive disclosure.

### ADR-6: Existing-test preserve gate
**Decisione:** Phase-5 Step 0 (pre-write): `if [ -f $TEST_PATH ]: skip_with_decision PRESERVE_EXISTING`. Log in Block 9.
**Rationale:** brownfield repo (16 test esistenti in Agent-B target) rischiano overwrite silente. Violazione "never destroy user work".

### ADR-7: Coverage probe-first quando esistono test
**Decisione:** se `stack.existing_test_frameworks != []` AND `module_coverage == []`, esegui Phase-6 probe (run test esistenti + parse coverage) PRIMA di Phase-5. Popola `module_coverage` → D1 attiva TIER-FIRST.
**Rationale:** primo batch ordering corretto; risparmio ~8K token su P-TIER fallback.

### ADR-8: gh variable → hint (non source of truth)
**Decisione:** `pre_existing_coverage_pct` viene da local report (lcov.info/jacoco.xml/coverage-summary.json) prima di tutto. `gh variable get TEST_COVERAGE_PERCENTAGE` retrocede a `pre_existing_coverage_hint` (auxiliary field).
**Rationale:** auth-state-dependent value viola determinismo; repo con healthy local coverage ma var stale = no-op.

### ADR-9: Drop `jq` dep (Phase 0+6)
**Decisione:** sostituire 3 chiamate `jq` con `python3 -c "import json; ..."`. `python3` e' gia' dep obbligatoria.
**Rationale:** rimuove l'unico vero forced user-prompt path ("install jq before invoking").

### ADR-10: Maven JaCoCo plugin pre-check
**Decisione:** in `select_command._resolve_jacoco_report_path`, grep pom per `<artifactId>jacoco-maven-plugin</artifactId>`. Se assente, emit error con snippet completo da incollare in `<plugins>`:
```xml
<plugin>
  <groupId>org.jacoco</groupId>
  <artifactId>jacoco-maven-plugin</artifactId>
  <version>0.8.12</version>
  <executions>
    <execution>
      <id>prepare-agent</id>
      <goals><goal>prepare-agent</goal></goals>
    </execution>
    <execution>
      <id>report</id>
      <phase>test</phase>
      <goals><goal>report</goal></goals>
    </execution>
  </executions>
</plugin>
```
**Rationale:** 3/4 SIAE Spring Boot pom dichiarano `<jacoco.version>` ma non wire-il plugin; Phase 6 fallisce in modo opaco. Snippet include `<executions>` complete (prepare-agent + report) per drop-in ready.

### ADR-11: `eval $COV_CMD` hardening
**Decisione:** `bash -c "$COV_CMD"` con `printf '%q'` shlex-quote del path repo prima della sostituzione.
**Rationale:** SKILL.md `cd "<repo>"` e' un placeholder LLM-substituted; metacharacter nel path = command injection.

### ADR-12: `detect_test_infrastructure` co-presence fix
**Decisione:** richiedi presenza di almeno 1 file `*.test.*`/`test_*.py`/`*Test.java`/`*Test.kt` dentro la directory PRIMA di contarla come test dir.
**Rationale:** Agent-D ha rilevato `src/api/test/` (route directory `/test`) classificato come test infrastructure. False-positive.

## 5. Flusso dati post-fix

```
Phase 0 (init)
  └─ check python3 only (jq dropped)
Phase 1 (parallel discover)
  ├─ detect_stack.py → stack.json (now includes orchestration_only, manifest_root)
  ├─ estimate_size.py → size.json (file_list with anchored P1 + codegen skip)
  └─ validate_env.py → env.json (uses manifest_root)
Gate-A (Orchestration)
  └─ if orchestration_only → Block 4 + END
Gate-B (Already-covered)
  └─ if pre_existing_coverage_pct ≥ 70 → Block 8 + END
Phase 2 (strategy) → references/phase-2-strategy.md
Phase 3 (sizing) if LARGE → batch-plan.json
Phase 4 (env) → install commands at $manifest_root
Gate-C (Probe-first)
  └─ if existing_test_frameworks != [] AND module_coverage == []
     → run Phase-6 PROBE first → populate module_coverage
Phase 5 (generation)
  ├─ Step 0: PRESERVE_EXISTING gate ([ -f $TEST_PATH ])
  ├─ Step 1+: D1 ordering (TIER-FIRST se module_coverage popolato)
  └─ template-cache.sh (unchanged)
Phase 6 (coverage)
  └─ lib/phase6-coverage.sh: cd $manifest_root && bash -c $cov_cmd (shlex-quoted)
Phase 7 (repair) → references/phase-7-repair.md
```
