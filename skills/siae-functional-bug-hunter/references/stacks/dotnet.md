# Stack: .NET / C#

## Stack id

`dotnet`

## Manifest fingerprints

- File globs: `**/*.csproj`, `**/*.fsproj`, `**/*.vbproj`, `**/*.sln`, `**/global.json`, `**/Directory.Build.props`, `**/Directory.Packages.props`, `**/*.cs`, `**/*.fs`, `**/*.vb`
- Content patterns: `<Project Sdk="Microsoft.NET.Sdk.Web">` or similar in `*.csproj`; `<TargetFramework>net8.0</TargetFramework>`.
- Negative match: a `Program.fs` with FSharp-only project → still dispatched here (F# treated under the .NET umbrella in v1).

## Analysis-unit granularity

- **.sln file**: each referenced `.csproj` is one analysis unit.
- **Multi-project repo without solution**: each `.csproj` is one unit.
- **Single project**: the `.csproj` directory is one unit.
- See [../repo_granularity.md](../repo_granularity.md).

## Parser

- Tree-sitter grammar: `tree-sitter-c-sharp`.
- Max AST depth: 5.
- F# uses regex fallback (no widely-deployed tree-sitter-fsharp grammar in v1).

## Entry-point kinds detected

| Framework / surface | `entry_point.kind` | Detection signal |
|---|---|---|
| ASP.NET Core controller | `http-route` | `[ApiController]` + `[Route("<path>")]` + `[HttpGet/Post/...]` |
| ASP.NET Core minimal APIs | `http-route` | `app.MapGet("<path>", handler)` / `app.MapPost(...)` in `Program.cs` |
| ASP.NET Core SignalR hub | `event-publisher` | class extending `Hub` with `public Task <Method>()` |
| gRPC service | `grpc-method` | class extending `<Service>.<Service>Base` generated by `protobuf-net.Grpc` or `Grpc.AspNetCore` |
| Azure Functions | depends on trigger | `[FunctionName("<n>")]` + trigger attributes (`[HttpTrigger]`, `[QueueTrigger]`, `[TimerTrigger]`, `[ServiceBusTrigger]`, `[EventGridTrigger]`) |
| AWS Lambda .NET | depends on event | `LambdaSerializer` + `public APIGatewayProxyResponse FunctionHandler(...)` |
| Hosted Service (`IHostedService`) | `scheduled-job` | class implementing `IHostedService` + `ExecuteAsync` |
| Quartz.NET job | `scheduled-job` | class implementing `IJob` + `Execute(IJobExecutionContext)` |
| MassTransit consumer | `message-consumer` | class implementing `IConsumer<T>` |
| NServiceBus handler | `message-consumer` | class implementing `IHandleMessages<T>` |
| MAUI / WinForms / WPF view | `ui-screen` | XAML with code-behind class + navigation registration |
| CLI entry | `cli-command` | `static int Main(string[] args)` + `System.CommandLine` registrations |

## Inputs typing

- ASP.NET Core: `[FromBody] T dto`, `[FromQuery] T q`, `[FromRoute] string id`, `[FromHeader(Name="...")]` → strongly typed; `T` is the resolved DTO with `DataAnnotations` validation.
- Minimal APIs: parameter binding inferred from delegate signature.
- Azure Functions: trigger parameter type (e.g. `string queueItem` for `[QueueTrigger]`) is the input.
- FluentValidation validators are captured as `inputs[].validation`.

## Side-effect detection

- Persistence: Entity Framework Core `_context.<Entity>.Add/Update/Remove` + `SaveChanges`, Dapper `Execute` with non-SELECT.
- HTTP clients: `HttpClient.GetAsync/PostAsync`, `IHttpClientFactory`-created clients.
- Message publishers: `IBus.Publish` (MassTransit), `IBusSession.Publish` (NServiceBus), Azure Service Bus `ServiceBusSender.SendMessageAsync`, AWS SDK `AmazonSQSClient.SendMessageAsync`.
- Filesystem and blob storage: `File.WriteAllTextAsync`, `BlobClient.UploadAsync`, AWS S3 `PutObjectAsync`.

## Cross-stack bridge hints

- `HttpClient` base addresses configured via `services.AddHttpClient<TClient>(c => c.BaseAddress = ...)` → `http-route` resolution.
- AWS / Azure SDK resource names → `aws-serverless` / Azure resource lookup (Azure not in scope, recorded as `external-unresolved`).
- Service Bus topic / queue names → cross-unit boundary.
- See [../cross_stack_bridges.md](../cross_stack_bridges.md).

## Bug-patterns row pointer

See [../bug_patterns.md](../bug_patterns.md) — rows where column `dotnet`
= `MUST-if-applicable`. Specifically: `NullReferenceException` on
nullable reference type ignored, async/await missing `.ConfigureAwait`
causing UI deadlock, EF Core `Include` missing for navigation property
read on disposed context, `DateTime.Now` without `DateTimeKind.Utc`,
`decimal` vs `double` for money, model-binding silently dropping fields
without `[Required]`, deadlock on `.Result` / `.Wait()` in hosted
contexts.

## Empty-input branch

If a unit is detected as `dotnet` but **zero** entry points are
extracted (e.g. a class library `.csproj`), the unit is recorded in
`coverage.md` with skip reason `no-entry-points`. Test projects
(`*.Tests.csproj`, `xunit` / `nunit` references) are auto-excluded with
`out-of-scope`.
