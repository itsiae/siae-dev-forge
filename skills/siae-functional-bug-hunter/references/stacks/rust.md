# Stack: Rust

## Stack id

`rust`

## Manifest fingerprints

- File globs: `**/Cargo.toml`, `**/Cargo.lock`, `**/*.rs`
- Content patterns: `[package]` table in `Cargo.toml`; `fn main()` in `src/main.rs` or `[lib]` declaration.
- Negative match: `[workspace]` table → workspace root, split into member crates.

## Analysis-unit granularity

- **Cargo workspace** (`[workspace] members = [...]`): each member crate is one analysis unit.
- **Single crate**: the directory containing `Cargo.toml` is one unit.
- **Bazel `rust_binary`**: each target is one unit.
- See [../repo_granularity.md](../repo_granularity.md).

## Parser

- Tree-sitter grammar: `tree-sitter-rust`.
- Max AST depth: 5.
- Macros are NOT expanded; macro-driven route registrations (e.g. Rocket `#[get]`) are detected by attribute pattern, not by post-expansion AST.

## Entry-point kinds detected

| Framework / surface | `entry_point.kind` | Detection signal |
|---|---|---|
| Actix-web | `http-route` | `App::new().service(web::resource("<path>").route(web::get().to(handler)))` or `#[get("<path>")]` attribute |
| Axum | `http-route` | `Router::new().route("<path>", get(handler))` |
| Rocket | `http-route` | `#[get("<path>")]`, `#[post("<path>")]`, `#[put("<path>")]` |
| Warp | `http-route` | `warp::path!("<segments>")` + `.and(warp::get()).map(handler)` |
| Tonic gRPC | `grpc-method` | `impl <Service> for <ServerStruct>` where `<Service>` is generated by `tonic-build` |
| AWS Lambda Rust runtime | `http-route` / `message-consumer` / `scheduled-job` depending on event type | `lambda_runtime::run(service_fn(handler))` in `main()` |
| Lapin (AMQP) consumer | `message-consumer` | `channel.basic_consume("<queue>", ...)` + delegate handler |
| Tokio cron | `scheduled-job` | `tokio-cron-scheduler` `Job::new("<expr>", fn)` |
| Clap CLI | `cli-command` | `#[derive(Parser)]` + `Cli::parse()` in `main()` |
| Custom Tauri command | `cli-command` | `#[tauri::command]` attribute |

## Inputs typing

- Axum extractors: `Json<T>`, `Path<T>`, `Query<T>`, `State<S>`, `Extension<T>` → strongly typed; `T` is the resolved struct with `serde` derives.
- Actix: `web::Json<T>`, `web::Path<T>`, `web::Query<T>` analogous.
- Serde attribute introspection: `#[serde(rename, default, deserialize_with)]` captured as `inputs[].validation` hints.
- `Result<T, E>` return → expected response type is `T`; error variants from `E` are mapped to status codes via framework-specific `IntoResponse` impls.

## Side-effect detection

- Persistence: `sqlx` `query!` / `execute` macros (non-SELECT detected by SQL string analysis); `diesel` `insert_into` / `update` / `delete`; SeaORM `ActiveModel::insert`.
- HTTP clients: `reqwest::Client::get/post`, `hyper::Client::request`, `ureq::get/post`.
- Message publishers: `rdkafka` `BaseProducer::send` / `FutureProducer::send`, `aws-sdk-sqs` `SendMessage`, `aws-sdk-sns` `Publish`.
- Filesystem: `std::fs::write`, `tokio::fs::write`.
- Unsafe blocks → recorded as observability hints; not directly a functional bug, but flagged for cross-check with the bug-patterns row.

## Cross-stack bridge hints

- `reqwest::Client::get("<url>")` literal → `http-route` lookup across units.
- AWS SDK calls → `aws-serverless` resolution.
- `tonic::transport::Channel::from_static("<target>")` → `grpc-method` resolution.
- See [../cross_stack_bridges.md](../cross_stack_bridges.md).

## Bug-patterns row pointer

See [../bug_patterns.md](../bug_patterns.md) — rows where column `rust` =
`MUST-if-applicable`. Specifically: `.unwrap()` on `Result` / `Option` in
request path (panic = 500 to client), integer overflow with `as` cast,
`Mutex` poisoning on panic, `Arc` reference-cycle memory growth as a
functional regression (slow OOM), async cancellation losing in-flight
work, `Drop` order with side effects, type inference picking unexpected
numeric type (i32 default).

## Empty-input branch

If a unit is detected as `rust` (`Cargo.toml` with `[package]`) but
**zero** entry points are extracted (e.g. a pure library crate or a
proc-macro crate), the unit is recorded in `coverage.md` with skip
reason `no-entry-points`. The unit still contributes to
`dependency_closure.md` when other units depend on it.
